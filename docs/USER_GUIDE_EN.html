Alerts — Active Threats

Central workstation for managing all active threats — Investigation, Approval, Collaboration

Security
identitymonitor.modernendpoint.tech/alerts
⚡ Alerts

🎯 Advanced Filtering

Filter by Status (Open / Assigned / Resolved) and Severity (Critical / High / Medium). Free-text search by user, IP, country, or application name.

🏷️ Automatic Tagging

Every alert receives a Severity badge (CRITICAL / HIGH / MEDIUM) and a Confidence badge (HIGH / MEDIUM CONFIDENCE) automatically based on the detection algorithm.

👤 Assign & Ownership

Assign an alert to a specific team member. Track ownership and accountability. Change ownership directly from within the alert panel.

✅ Approval Workflow

Critical actions (Wipe, Block, Reset Password) go through an Approval flow — Approved, Rejected, or assigned for closure.

  • 🔍
    Click an alert — opens the Investigation panel with Timeline, Raw data, Evidence and Recommended actions
  • 🙋
    Assign to me — takes ownership of an open alert
  • ✅
    Approve / Reject — approve or reject a pending action
  • 📌
    Acknowledge — marks the threat as reviewed and moves it to Documented status
  • 🔄
    Refresh — reloads the alert queue from the backend
  • 👑
    Role owner — top-right button — adds Role-based ownership across the entire tenant
💡 An alert tagged HIGH CONFIDENCE was triggered by multiple simultaneous indicators (unusual IP + new country + unrecognised device). These require immediate attention.

Exposure Center

Track at-risk identities, over-privileged accounts and blast radius for every privileged account

Security
identitymonitor.modernendpoint.tech/users
👥 Exposure

🎯 Privileged Identity Watchlist

A table of all high-privilege identities, sorted by Risk Score. Shows Roles, Account state, and Open threats per user.

💥 Blast Radius

Automatic calculation — how much damage could a user cause if their account were compromised. Includes a Role spread map and Suggested next steps.

🏥 Hygiene Score

Per-identity hygiene score: MFA, Conditional Access, JIT status, Last sign-in. Identifies accounts an attacker could use immediately.

🎨 Risk Colour Coding

CRITICAL (red) / HIGH (orange) / MEDIUM (yellow) / CLEAN (green). Click a row to open a full detail panel on the right.

  • 🔍
    Search — search by name, UPN or Role
  • 👁️
    Click a row — opens the "Exposure Drill" panel with threat history, role assignment and hygiene signals
  • 📊
    Ordered by risk — sorted by descending Risk Score
⚠️ A user with CRITICAL risk + Enabled account + Global Admin is an immediate threat. Apply a Conditional Access Policy or perform an urgent MFA reset.

Vulnerability Remediation

Close vulnerabilities directly from the portal — no need to switch to another tool

Security
identitymonitor.modernendpoint.tech/remediation
🛠️ Remediation

📋 Vulnerability List

All CVEs and findings from Defender, with filters by: CVE ID, Product, Publisher, Category, Severity. Includes "Remediation required only" and "Exposed devices only" toggles.

🗂️ 3 Tabs per Finding

Details — CVE info, severity, description. Exposed Devices — affected machines. Remediation Plan — execution plan.

🤖 4 Executor Types

Webapp — patch applications (Edge, Chrome, 7-Zip) via external service. Windows Update — OS updates. Intune Policy — push Intune config. Script — PowerShell remediation.

⚡ One-Click Execute

Click "Plan remediation" → the system determines the optimal execution path. Click "Execute remediation" → runs the fix. Result shown with ✅ / ❌ and detailed status.

  • 🔍
    Filter — by CVE ID, Product, Publisher, Category, Severity
  • 📱
    Exposed devices tab — list of affected devices with Device ID and Device Name
  • 📝
    Plan remediation — calculates the automatic execution path based on finding type
  • ▶️
    Execute remediation — runs the fix (only appears when the service is connected and ready)
  • 🔄
    Refresh plan — recalculates the execution plan
  • 📌
    Admin Consent button — appears when Intune permissions are missing — click to request Admin Consent
💡 Applications such as Microsoft Edge, Chrome, 7-Zip and Firefox are automatically routed to the Webapp Executor — an external service at app.modernendpoint.tech that identifies and patches them without additional configuration.

PIM Security Analysis

Privileged Identity Management analysis — detect Permanent Admins, missing JIT, over-privileged accounts

Security
identitymonitor.modernendpoint.tech/pim
🔑 PIM Analysis

🎓 PIM Score

A–F grade based on the organisation's PIM posture. D = Poor. Composed of: Permanent Roles, JIT Eligible assignments, Global Admin count, PIM Enabled status.

🚨 Findings

Specific findings with severity: "PIM Just-in-Time is not configured", "2 permanent Global Administrator assignments" — each with remediation instructions.

💡 Recommendations

Actionable recommendations with exact paths: "Enable PIM → Entra Admin Center → Identity Governance → Privileged Identity Management → Discover". Labelled Quick Win or Some effort.

🛡️ Risk Labels

Each recommendation is tagged CRITICAL / HIGH + RISK, with a clear explanation of why it matters ("JIT access eliminates 80%+ of insider threat").

  • 📋
    Findings — list of existing issues, each with a Remediation Path button
  • 💡
    Recommendations — prioritised remediation steps with Quick Win marking
  • 🔄
    Last analyzed — shows when the last analysis ran + Refresh button
⚠️ PIM Score D (Poor) = all Admin assignments are Permanent with no JIT. An attacker who gains access has permanent privileges with no time limit. Enable Entra ID P2 and configure Eligible assignments.

🌐 MSP Fleet

🏢 All Tenants View

A card for each connected tenant showing: Critical Alerts, Open Alerts, Need Action, All Clear. Sorted by Highest Risk by default.

💡 MSP Fleet is designed for Managed Service Providers managing multiple customers. Each customer connects once via Admin Consent and is then monitored automatically.

Settings

Per-tenant configuration: Detection rules, Notifications, Admins, Approvals, SIEM integrations

Management
identitymonitor.modernendpoint.tech/settings
⚙️ Settings

💳 Plan & Trial

Shows current plan and Trial expiry. Notification Center snapshot with Unread / Approval / Mentions / Escalations counters.

🔍 Detection Rules

Tune detection sensitivity: Impossible Travel, New Country, Unrecognised Device, Suspicious App. Each rule can be enabled / disabled and have its threshold adjusted.

⚡ Auto-Actions

Automated responses by severity: Block user, Reset MFA, Require re-auth, Send Telegram alert. Configure trigger conditions per action.

🔔 Notifications

Configure recipients: Email addresses, Telegram Chat IDs. Select notification triggers by severity and event type.

👥 Admins

Manage who can access the platform and at what permission level (SOC Analyst / Security Manager / Admin).

✅ Automation & Approvals

Define which actions require two-step Approval before execution (Wipe, Block, Password Reset). Configure the Approvers list.

🚫 Whitelist

Add IPs, Users and Countries to the Whitelist — they will not generate alerts. Useful for MSP IPs and travelling executives.

🏢 Business Hours

Define business hours — alerts triggered outside those hours automatically receive Higher severity.

📡 SIEM & Log Analytics

Connect to an external SIEM (Sentinel, Splunk): Webhook URL, API Key, Log format (CEF / JSON).

    Plan & Trial — first tab — plan status and notification counters
  • 🔍
    Detection — tune detection rules and sensitivity thresholds
  • ⚡
    Auto-Actions — automated actions per trigger
  • 👥
    Admins — manage platform access
  • 🔔
    Notifications — manage recipients
  • ✅
    Automation & Approvals — Approval workflows
  • 🚫
    Whitelist — trusted IPs / Users / Countries
  • 🕒
    Business Hours — monitoring hours configuration
  • 📡
    SIEM & Log Analytics — external SIEM connection
