Alerts — Active Threats

תחנת עבודה מרכזית לניהול כל האיומים הפעילים — Investigation, Approval, Collaboration

Security
identitymonitor.modernendpoint.tech/alerts
⚡ Alerts

🎯 סינון מתקדם

סינון לפי Status (Open/Assigned/Resolved) ו-Severity (Critical/High/Medium). חיפוש חופשי לפי משתמש, IP, מדינה, אפליקציה.

🏷️ תיוג אוטומטי

כל alert מקבל Severity badge (CRITICAL/HIGH/MEDIUM) ו-Confidence badge (HIGH/MEDIUM CONFIDENCE) אוטומטית לפי אלגוריתם הזיהוי.

👤 Assign & Ownership

הקצאת alert לחבר צוות ספציפי. מעקב אחר בעלות ואחריות. שינוי בעלות מתוך ממשק ה-alert.

✅ Approval Workflow

פעולות קריטיות (Wipe, Block, Reset Password) עוברות Approval flow — מאושר, נדחה, או מוקצה לסגירה.

  • 🔍
    לחיצה על alert — פתיחת פאנל Investigation עם Timeline, Raw data, Evidence ו-Recommended actions
  • 🙋
    Assign to me — לוקח בעלות על alert פתוח
  • ✅
    Approve / Reject — אישור/דחייה של פעולה שדורשת Approval
  • 📌
    Acknowledge — מאשר שהאיום נבדק, מעביר לסטטוס Documented
  • 🔄
    Refresh — טעינה מחדש של התור מה-backend
  • 👑
    Role owner — כפתור ימין עליון — מוסיף Role-based ownership לכל ה-tenant
💡 Alert עם badge HIGH CONFIDENCE = זוהה על ידי מספר אינדיקטורים בו-זמנית (IP חריג + מדינה חדשה + מכשיר לא מוכר). כאלה מחייבים טיפול מיידי.

👥 Exposure

🎯 Privileged Identity Watchlist

טבלה של כל הזהויות בעלות הרשאות גבוהות, ממוינת לפי Risk Score. מציגה Roles, Account state, Open threats.

💥 Blast Radius

חישוב אוטומטי — כמה נזק יכול משתמש לגרום אם חשבונו נפרץ. כולל מפת Role spread ו-Suggested next steps.

🏥 Hygiene Score

ציון היגיינה לכל זהות: MFA, Conditional Access, JIT status, Last sign-in. מזהה חשבונות שיכולים להיות בשימוש מיידי על ידי תוקף.

🎨 Risk Color Coding

CRITICAL (אדום) / HIGH (כתום) / MEDIUM (צהוב) / CLEAN (ירוק). לחיצה על שורה פותחת פאנל פירוט מלא ימינה.

  • 🔍
    חיפוש — חיפוש לפי שם/UPN/Role
  • 👁️
    לחיצה על שורה — פאנל "Exposure Drill" עם threat history, role assignment ו-hygiene signals
  • 📊
    Ordered by risk — מיון לפי Risk Score יורד
⚠️ משתמש עם CRITICAL risk + Enabled account + Global Admin = סיכון מיידי. יש להפעיל Conditional Access Policy או לבצע MFA reset דחוף.

Vulnerability Remediation

סגירת vulnerabilities ישירות מהממשק — ללא כניסה לפורטל אחר

Security
identitymonitor.modernendpoint.tech/remediation
🛠️ Remediation

📋 Vulnerability List

רשימת כל ה-CVEs ו-findings מ-Defender, עם סינון לפי: CVE ID, Product, Publisher, Category, Severity. כולל Remediation required only ו-Exposed devices only.

🗂️ 3 טאבים לכל Finding

Details — פרטי ה-CVE, חומרה, תיאור. Exposed Devices — מכשירים מושפעים. Remediation Plan — תוכנית ביצוע.

🤖 4 Executor Types

webapp — תיקון אפליקציות (Edge, Chrome, 7-Zip) דרך שירות חיצוני. Windows Update — עדכוני מערכת. Intune Policy — מדיניות Intune. Script — PowerShell Remediation.

⚡ One-Click Execute

לחיצת "Plan remediation" → בחינת הנתיב האוטומטי. לחיצת "Execute remediation" → ביצוע בפועל. תוצאה מוצגת עם ✅ / ❌ וסטטוס מפורט.

  • 🔍
    סינון — CVE ID, Product, Publisher, Category, Severity
  • 📱
    Exposed devices tab — רשימת מכשירים חשופים עם Device ID ו-Device Name
  • 📝
    Plan remediation — מחשב נתיב ביצוע אוטומטי לפי סוג ה-finding
  • ▶️
    Execute remediation — מבצע את התיקון בפועל (מופיע רק כשהשירות מחובר ומוכן)
  • 🔄
    Refresh plan — חישוב מחדש של תוכנית הביצוע
  • 📌
    Admin Consent button — מופיע אם חסרות הרשאות Intune — לוחץ לבקשת Admin Consent
💡 אפליקציות כגון Microsoft Edge, Chrome, 7-Zip, Firefox מנותבות אוטומטית ל-Webapp Executor — שירות חיצוני ב-app.modernendpoint.tech שמזהה ומתקן ללא צורך בהגדרות נוספות.

PIM Security Analysis

ניתוח Privileged Identity Management — זיהוי Permanent Admins, חסר JIT, הרשאות יתר

Security
identitymonitor.modernendpoint.tech/pim
🔑 PIM Analysis

🎓 PIM Score

ציון A–F לפי מצב ה-PIM בארגון. D = Poor. מורכב מ: Permanent Roles, JIT Eligible, Global Admins, PIM Enabled status.

🚨 Findings

ממצאים ספציפיים עם חומרה: "PIM Just-in-Time is not configured", "2 permanent Global Administrator assignments" — עם הוראות תיקון.

💡 Recommendations

המלצות עם Path מדויק: "Enable PIM → Entra Admin Center → Identity Governance → Privileged Identity Management → Discover". כולל Quick Win / Some effort labeling.

🛡️ Risk Labels

כל המלצה מסומנת: CRITICAL/HIGH + RISK. כולל הסבר מדויק למה זה סיכון ("JIT access eliminates 80%+ of insider threat").

  • 📋
    Findings — רשימת בעיות קיימות עם לחצן Remediation Path לכל אחת
  • 💡
    Recommendations — צעדי תיקון מדורגים לפי עדיפות עם Quick Win marking
  • 🔄
    Last analyzed — הצגת מתי בוצע הניתוח האחרון + Refresh
⚠️ PIM Score D (Poor) = כל ה-Admin assignments הם Permanent ללא JIT. תוקף שמשיג גישה → יש לו הרשאות קבועות ללא חלון זמן. יש להפעיל Entra ID P2 ולהגדיר Eligible assignments.

⚙️ Settings

💳 Plan & Trial

🔍 Detection Rules

כיוון רגישות זיהוי: Impossible Travel, New Country, Unrecognized Device, Suspicious App. כל כלל ניתן לפעיל/כיבוי ולהגדרת Threshold.

⚡ Auto-Actions

פעולות אוטומטיות לפי חומרה: Block user, Reset MFA, Require re-auth, Send Telegram alert. הגדרת תנאי הפעלה.

🔔 Notifications

הגדרת נמענים: Email recipients, Telegram Chat IDs. בחירת Notification triggers לפי severity וסוג אירוע.

👥 Admins

ניהול מי יכול לגשת לפלטפורמה ובאיזו רמת הרשאה (SOC Analyst / Security Manager / Admin).

✅ Automation & Approvals

הגדרת אילו פעולות דורשות Approval דו-שלבי לפני ביצוע (Wipe, Block, Password Reset). הגדרת Approvers list.

🚫 Whitelist

הוספת IPs, Users, Countries ל-Whitelist — לא יייצרו alerts. שימושי עבור MSP IPs, Traveling executives.

🏢 Business Hours

הגדרת שעות עסקים — alert בשעות לא עסקיות מקבל Higher severity אוטומטית.

📡 SIEM & Log Analytics

חיבור ל-SIEM חיצוני (Sentinel, Splunk): Webhook URL, API Key, Log format (CEF/JSON).
